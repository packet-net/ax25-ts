## [0.13.0] — 2026-06-12

Parity with the C# reference at packet.net **lib-v0.8.0**: two session-quirk figure-defect fixes and one recorded transport exception.

### Added

- **`ax25Spec13ClampSrejWindowToHalfModulus` session quirk** ([#62](https://github.com/M0LTE/ax25-ts/pull/62)) — mirrors [packet.net#394](https://github.com/m0lte/packet.net/pull/394) ([packethacking/ax25spec#13](https://github.com/packethacking/ax25spec/issues/13)). Selective Repeat needs the textbook `2·W ≤ modulus` bound, but AX.25 lets `k` range to `modulus−1` and the figures never enforce the tighter limit, so above `modulus/2` two in-flight frames can share an N(S) and the receiver can deliver a stale stored I-frame — **silent payload corruption** under SREJ + loss. The quirk (default on; off under `strictlyFaithful`) caps **both** the send window and the receive-side out-of-window-discard bound at `modulus/2` whenever SREJ is enabled, via `Ax25SessionContext.effectiveWindow`. Go-back-N (SREJ off) is never capped — it tolerates `k` up to `modulus−1`; the mod-8 default `k=4 = modulus/2` is exactly at the limit and unchanged.
- **`ax25Spec9AckProgressResetsRc` session quirk** ([#60](https://github.com/M0LTE/ax25-ts/pull/60)) — mirrors [packet.net ax25spec#9](https://github.com/packethacking/ax25spec/issues/9). The figures only reset the retry counter RC on a fully-acked checkpoint, so under sustained progressing-but-lossy transfer a handful of T1 hiccups ratchet RC to N2 and the sender spontaneously DMs a live session. The quirk (default on; off under `strictlyFaithful`) clamps RC to 1 at a T1 expiry that follows V(A)-advancing progress, so RC counts *consecutive* recovery failures and a genuinely dead link still exhausts N2. Deliberately at-expiry, not an eager zero at ack time (RC==0 doubles as the Karn sampling signal).

### Changed

- **Recorded parity exception for `Ax25ListenerOptions.RestartT1OnTxComplete`** ([#61](https://github.com/M0LTE/ax25-ts/pull/61)) — packet.net's `kiss.t1FromTxComplete` re-arms a running T1 from the TNC's ACKMODE TX-completion echo. The echo is a TNC/node-host transport capability and no TS transport implements `sendFrameWithAck` today, so the option would be inert here; the drift guard carries a reviewed exception in `scripts/parity-exceptions.json` until an ACKMODE-capable TS transport exists.

## [0.11.0] — 2026-06-08

### Added

- **INP3 connector live host wiring — the overlay is now a turnkey feature on `NetRomConnector` (TS↔C# parity, default-off)** — the TS analogue of the C# `NetRomService` host-integration ([packet.net#337](https://github.com/M0LTE/packet.net/pull/337)/[#338](https://github.com/M0LTE/packet.net/pull/338)), completing the previous slice's host-free layers into a live feature. When `NetRomConnectorOptions.inp3` is supplied (a shared writable `NetRomRoutingTable` + the INP3 knobs), the connector owns an `Inp3Engine` + `Inp3UpdateScheduler` and: (1) **taps inbound PID-0xCF for RIF / L3RTT ahead of the L4 path** (`dispatchInp3`, mirroring the C# precedence — a `0xFF`-led frame is a RIF, consumed whether or not it parses; an L3RTT to `L3RTT-0` is timed/reflected; anything else falls through to the existing circuit/forward dispatch); (2) **probes** interlink neighbours and **ingests** their RIFs into the shared table as time-routes (so NODES quality-routes and RIF time-routes coexist on one table — the connector exposes the writable handle via the new `NetRomService.routingTable` getter); (3) **fans out** poison-reversed RIFs, with the recently-withdrawn set **drained once per round** (the atomic round boundary that mirrors the C# host-thread race fix — a concurrent mid-round ingest lands in the next round, never cleared unadvertised); (4) on the 180 s reflection-reset of an INP3-capable neighbour, drops its routes (`markNeighbourDown`) + signals the embedder's failover hook; and (5) when `preferInp3Routes` is set, **forwards / connects over the lowest-target-time route** (`decideForward` + the `connect`/reply next-hop now resolve through `selectActiveRoute`). The engine/scheduler are driven by the connector's existing `tick()` (no ambient timer). **Default-off**: omit `inp3` and the connector is byte-for-byte today (the `inp3*` fields stay null). **Tests:** `tests/netrom/connector-inp3.test.ts` drives the live loop through a controllable fake interlink — probe-on-tick, SNTT measurement, RIF ingest → time-route, the periodic poison-reversed RIF fan-out, RIF-consumed-never-forwarded, and the default-off guarantee. 1166 unit tests green; typecheck + examples + build clean.

- **INP3 time-based routing overlay — the host-free protocol layers + dual-metric forwarding (TS↔C# parity, default-off)** — the TS cascade of the merged C# INP3 reference ([packet.net#333](https://github.com/M0LTE/packet.net/pull/333)/[#334](https://github.com/M0LTE/packet.net/pull/334)/[#335](https://github.com/M0LTE/packet.net/pull/335)/[#336](https://github.com/M0LTE/packet.net/pull/336) + host-integration [#337](https://github.com/M0LTE/packet.net/pull/337) + forwarding-by-time [#338](https://github.com/M0LTE/packet.net/pull/338)). INP3 (Andreas Gal DB7KG / NORD><LINK) is a time-based, triggered NET/ROM routing overlay that rides the same PID-0xCF interlink as vanilla NET/ROM, stays drop-in compatible, and is **default-off** — a node behaves exactly as today until the overlay is enabled. This slice lands the **host-free protocol layers** (the TS analogue of the C# library slices I-1..I-4) plus the dual-metric route model and forwarding-by-time capability; the live host wiring (engine/scheduler ownership + the inbound RIF/L3RTT dispatch on the connector's interlink) is the embedder/connector slice landing next, the TS analogue of the C# `NetRomService` host-integration. **Wire codecs** (`src/netrom/inp3-l3rtt.ts`, `inp3-rif.ts`): `Inp3L3RttFrame` (the L3RTT probe/reflect frame — a `NetRomPacket` to dest `L3RTT-0`, opcode 0x02, `$N`/`$IX` capability text; a reflection is recognised by origin == self) and the `0xFF`-signed routing-information frame (`Inp3Rif`/`Inp3Rip`/`Inp3Tlv` — a RIP is `[7-byte shifted callsign][1-byte hop][2-byte MSB-first 10 ms target time; 0xEA60 = horizon = withdrawal][TLVs][0x00 EOP]`, alias/IP TLVs decoded with unknown types retained verbatim, plus the load-bearing forced EOP-vs-alias disambiguation). Both **total/fuzz-safe** and **byte-parity-verified** against the locked hex vectors in `packet.net/docs/netrom-inp3-i1-wire-spec.md`, with `INP3_PARSE_STRICT`/`LENIENT`/`BPQ`/`XROUTER` presets. **Link timing** (`src/netrom/inp3-sntt.ts`, `inp3-engine.ts`, `inp3-options.ts`): `Inp3Sntt`, the integer SNTT IIR smoother over `RTT/2` samples (a 1/8-gain rounded EWMA, integer-only so the three stacks agree bit-for-bit), and `Inp3Engine`, the host-free probe/reflect engine (mirrors `CircuitManager`: injected `now` + embedder-driven `tick()`, no ambient timer) that probes via a `sendL3Rtt` sink, folds reflected `RTT/2` into the smoother, learns `$N`/`$IX` capability, and fires `onNeighbourDown` on the 180 s reset — **only for a previously-INP3-capable neighbour** (a never-capable vanilla neighbour is dropped silently, never torn down: graceful degradation). **Dual-metric routing** (`src/netrom/routing-table.ts`, `inp3-route-selector.ts`): `NetRomRoute` gains an optional `Inp3RouteMetric` (target time + hop) so a destination holds NODES quality-routes and RIF time-routes at once; `ingestRif` upserts time-routes (`localTargetTime = peerTarget + neighbourSNTT + 10 ms/hop`; at/above-horizon = withdraw; unmeasured link learns nothing) without disturbing quality ingestion; `buildRif` advertises every held time-route with **poison-reverse over the full kept-route set** plus one-shot horizon withdrawals from an atomically-drained recently-withdrawn set (`drainRecentlyWithdrawn`); and the pure `selectActiveRoute(dest, preferInp3Routes)` picks the lowest-target-time route when the knob is on (else best-quality — verified to degenerate byte-for-byte to today when off). **Forwarding-by-time** (`src/netrom/forwarding.ts`): `decideForward` gains a `preferInp3Routes` parameter — forward over the lowest-target-time INP3 route (way-it-came excluded, quality fallback when none usable); per-flow LB stays quality-space (the time-space always forwards the single fastest path). **Triggered updates** (`src/netrom/inp3-update-scheduler.ts`): `Inp3UpdateScheduler` — negative-immediate / positive-batched(debounce) / periodic full RIF, emitting per-neighbour advertise intents. **Tests:** a faithful port of every C# `Inp3*Tests` to vitest (the SNTT golden trajectories + band assertions, the exact §1.5 hex parity vectors + totality sweeps, the engine probe/reflect/reset incl. the never-probed-vs-clock-0 sentinel + the vanilla-neighbour-never-torn-down guard, the dual-metric coexistence + poison-reverse + atomic drain, the scheduler, and forwarding-by-time). The whole host-free surface is exported from the package index. **Built by an agent fleet against the merged C# reference as the oracle, then hand-integrated + byte-parity-verified.** **Live host wiring:** the engine/scheduler driven on a real interlink, the inbound RIF/L3RTT dispatch ahead of the L4 path, and the connector consuming `preferInp3Routes` + `selectActiveRoute` all landed in this same release — see the INP3 connector live-wiring entry above.

- **NET/ROM `connect <alias>` outbound routing — open an end-to-end L4 circuit across the network to a node by name (TS↔C# parity)** — the integration slice (TS-4) that ties the L4 circuit FSM (`NetRomCircuit` / `CircuitManager`, the previous slice), the learned routing table (`NetRomRoutingTable`), and the `Ax25Listener` together into an end-to-end NET/ROM connect. The TS parity leg of the C# `Packet.Node.Core.NetRom.NetRomOutboundConnector` + `NetRomNodeConnection` + the L4-circuit/interlink portion of `NetRomService` ([packet.net#312](https://github.com/M0LTE/packet.net/pull/312)). **New `src/netrom/connector.ts` (`NetRomConnector`)** — given a destination **alias** (e.g. `SOT`) or **callsign** (e.g. `GB7SOT`), it (1) resolves the best route in the routing table (the new `resolveDestination` / `neighbourFor` snapshot helpers in `src/netrom/routing-table.ts`, mirroring the C# `NetRomRoutingSnapshot.ResolveDestination` / `NeighbourFor`); on a routing miss it throws `NetRomNoRouteError` — the clean seam for the embedder to fall back to a direct same-port AX.25 dial (the fallback itself is the embedder's job, mirroring the C# connector deferring to its wrapped `fallback`); (2) opens a CONNECTED-mode AX.25 **interlink** (PID 0xCF) to the route's best **neighbour** via `Ax25Listener.connect`, reusing a live interlink to that neighbour if one exists (`ensureInterlink`); (3) runs a `NetRomCircuit` end-to-end over that interlink, with the L3 network header naming origin = our node, destination = the end node; and (4) drives the circuit's `Connect` (carrying the originating user + node via the Connect-Request-info codec) and presents the established circuit as a duplex connection. **The key integration seam** is bidirectional: the owned `CircuitManager`'s `sendPacket` sink routes each outbound `NetRomPacket` to its L3 destination's best-neighbour interlink and ships it as a PID-0xCF I-frame via `Ax25Listener.sendData` (`sendNetRomPacket`, with the C# `SendNetRomPacket` next-hop order: direct interlink to the destination node → best route → destination-as-neighbour), and every interlink session's inbound PID-0xCF DL-DATA indication is parsed into a `NetRomPacket` and fed to `CircuitManager.onPacket` (`tapInterlinkSession` / `onInterlinkData`, mirroring the C# `OnSessionAccepted` / `OnInterlinkData` tap, idempotent per session) — so the circuit layer (which speaks only datagrams, knowing nothing of AX.25) is bridged onto real connected-mode AX.25 sessions in both directions. An **inbound** circuit a remote opens to us raises `onIncomingConnection` (auto-accept + wrap, the data tap wired before the accept so no Info racing the connect-ack is lost — mirroring the C# `OnIncomingCircuit` + `RunInboundConsole`). **New `src/netrom/connection.ts` (`NetRomConnection`)** — the duplex byte stream over an L4 circuit (the network-routed analogue of `Ax25ListenerSession`): `onData` / `write` / `onClosed` + an awaitable `completion`, the event-driven TS shape mirroring the C# `NetRomNodeConnection`'s `Channel`-backed `ReadAsync` / `WriteAsync` / `Completion`. It **buffers inbound data that arrives before a reader attaches** then flushes in order (the TS analogue of the C# unbounded inbound channel — so a banner the far end sends the instant it accepts isn't lost to a late subscriber), tells the circuit a frame was consumed (`onDeliveryDrained`) for choke release, and on `dispose` disconnects the circuit + settles completion. Like the rest of the NET/ROM module the connector is **embedder-driven** (drive retransmits via `tick()` from a `setInterval`; the library owns no ambient timers) and **opt-in** (`enabled` defaults `false` — a node must opt in to open interlinks + circuits on the air, the C# `ConnectEnabled` gate). The whole surface is exported from the package index (`NetRomConnector`, `NetRomConnection`, `NetRomNoRouteError`, `resolveDestination`, `neighbourFor`, the option/source types). **Unit tests:** `tests/netrom/connector.test.ts` (a two-node in-memory harness — two real `Ax25Listener`s cross-connected over a `MockTransport` pair, a real `NetRomConnector` per node — where node A `connect`s by **alias** to an end node reachable via neighbour B: asserts the AX.25 interlink session opens to B, the `NetRomCircuit` reaches Connected on both ends, data round-trips end-to-end over the circuit, and a disconnect tears down both the circuit and the interlink — the unit mirror of the C# `NetRomL3L4IntegrationTests` two-node `connect <alias>` test; run by packet.net's `interop.yml` sibling-clone phase against its docker LinBPQ stack, not ax25-ts's unit-only CI):** `tests/integration/netrom-l4-connect-via-axudp.test.ts` — over the AXUDP transport to docker LinBPQ, originates NODES so BPQ learns + routes to us, hears BPQ's NODES so we have a route, opens a NET/ROM L4 circuit pdn→BPQ via `NetRomConnector.connect` and round-trips a `PORTS` command to BPQ's node prompt (Information both ways), disconnects, then drives BPQ's telnet `C <alias>` so BPQ originates a reverse circuit pdn's `onIncomingConnection` accepts + echoes — the AXUDP mirror of the C# `NetRomL4CircuitViaAxudp`, reusing the existing AXUDP/telnet interop helpers + the `B`-flagged MAP fixture. **Parity gaps (by design):** the C# `NetRomService` fuses ingest + NODES origination + the L4 connector + interlink management + console-bridge hooks + a `TimeProvider` sweep timer into one node-host type; ## [0.7.0] — 2026-06-05

### Added

- **AXUDP transport (`AxudpTransport`) + Tier-2 protocol interop over it — AX.25-frames-over-UDP / BPQAXIP, FCS-always (TS↔C# parity)** — a Node UDP-socket transport that plugs into the same `Ax25Transport` seam as `TcpKissTransport`, so the `Ax25Listener` + the NET/ROM module run over a BPQAXIP/UDP tunnel to a real AXIP/AXUDP peer (LinBPQ's BPQAXIP, XRouter, ax25ipd, JNOS) exactly as over KISS-TCP. The TS parity leg of the C# `Packet.Axudp` / `AxudpKissModem` ([packet.net#299](https://github.com/M0LTE/packet.net/pull/299)/[#301](https://github.com/M0LTE/packet.net/pull/301)) and the Tier-2 interop re-home ([packet.net#311](https://github.com/M0LTE/packet.net/pull/311)). **AXUDP is not KISS** — there is no SLIP framing, command byte, or CSMA: a datagram's payload *is* the AX.25 frame body (the KISS-form octets the listener produces) followed by the **2-octet AX.25 FCS** (CRC-16/X.25, low byte first). **The FCS is unconditional — there is no FCS-less mode.** A citation survey of every real AXIP/AXUDP implementation (RFC 1226 + rfc1226-bis, ax25ipd, LinBPQ's BPQAXIP, XRouter, JNOS) found the FCS mandatory in all of them and FCS-less accepted by none — so an AXUDP port talks to a real peer out of the box; an FCS-less form was a self-only invented format on the C# side and was removed ([packet.net#304](https://github.com/M0LTE/packet.net/pull/304)/[#306](https://github.com/M0LTE/packet.net/pull/306)) — this transport ships FCS-always and never grew that detour. Stripping + validating the FCS on receive is mandatory not cosmetic: the AX.25 parser rejects an S-frame (RR/RNR/REJ ack) carrying a trailing FCS tail, so an unstripped tail would silently drop every supervisory frame and break connected mode; a bad-FCS datagram is dropped (as a real peer does, BPQAXIP logs "Invalid CRC"). **New `src/fcs.ts`** — the CRC-16/X.25 FCS codec (poly 0x1021 / reflected 0x8408, init 0xFFFF, RefIn/RefOut, XorOut 0xFFFF; standard vector `"123456789"` → 0x906E; the BPQAXIP good residue 0xF0B8 anchored in tests), byte-for-byte the C# `Packet.Core.Crc16Ccitt` and LinBPQ's `bpqaxip.c` `compute_crc`, with `appendFcs` / `stripFcs` helpers. Plain KISS transports never touch it (the TNC owns HDLC framing + the FCS); it exists for AXUDP, which has no TNC. **New `src/axudp-transport.ts`** (`AxudpTransport`, Node-only, exported via the `@packet-net/ax25/axudp-transport` subpath — same browser-safety rationale as the TCP transport: `node:dgram` has no browser polyfill, so it stays off the main entrypoint). Binds a UDP socket on `start` (a configurable fixed `localPort` for a peer that dials us by a known port — e.g. a BPQAXIP `MAP` entry — or 0 for ephemeral; **all interfaces by default**, since a real peer behind a NAT/bridge originates back to the gateway address, not loopback), sends every outbound frame (FCS appended) as one datagram to the configured remote, and surfaces inbound datagrams (FCS stripped + validated, bad-FCS dropped) via the `onFrame` callback; point-to-point (every frame goes to the one remote, the peer ignores by address). The `node:dgram` slice it needs is added to the hand-written `src/node-types.d.ts` shim (the repo deliberately avoids `@types/node`). **Unit tests** — `tests/fcs.test.ts` (the CRC catalogue + single-byte bit-ordering vectors locking interop with the C# codec, append/strip round-trip, bad-FCS dropped, too-short dropped, the 0xF0B8 wire-compat residue, a standalone-buffer guard) and `tests/axudp-transport.test.ts` (a `MockUdpSocket` paired fake mirroring the TCP transport's `MockSocket`: FCS round-trip on send/receive, bad-FCS + too-short dropped, send-before-start throws, bind/listening, stop idempotency). **Integration tier (Tier 2 — frame-perfect, run by packet.net's `interop.yml` sibling-clone phase B against its docker stack):** `tests/integration/linbpq-via-axudp.test.ts` — `Ax25Stack` over `AxudpTransport` dials real LinBPQ over BPQAXIP/UDP (SABM/UA via BPQ's `AUTOADDQUIET`), exchanges I-frames (BPQ's banner + a `P\r` ports-command round-trip — exercising BPQ's RR S-frame acks, which an unstripped FCS tail would break), and disconnects; a successful FCS-bearing handshake is itself the FCS proof (the C# `FcsLess_Sabm_IsDropped` guard locks the negative case). `tests/integration/netrom-nodes-ingest-via-axudp.test.ts` — the read-only NET/ROM module ingests real LinBPQ's NODES over the UDP tunnel and learns it as a neighbour, the AXUDP analog of the net-sim NODES-ingest test, **deterministic in ~2 s** (vs the net-sim version's 90 s channel-loss budget) and **load-insensitive** (green across many consecutive runs incl. under sustained CPU saturation, load avg > 40 — no AFSK decode to glitch under contention, the whole point of shifting protocol coverage onto AXUDP). The net-sim protocol smokes (`linbpq-via-netsim`, `netrom-nodes-ingest-via-netsim`) are **kept** as the thin Tier-3 modem/channel smoke. (BPQAXIP delivers its own NODES over a point-to-point AXIP port only to a `B`-flagged `BROADCAST NODES` MAP recipient that it treats as a live neighbour; the fixture adds `MAP PNTSAX-1 … B` for our read-only listener and the test warms BPQ by sending its own NODES through the shared transport, since the netrom module is TX-free by design — see packet.net `docker/linbpq/bpq32.cfg` + docs/plan.md §7.0.) The transport is exported only via the `@packet-net/ax25/axudp-transport` subpath, so the main browser entrypoint never pulls `node:dgram` — the Web Serial build + downstream browser consumers are unaffected.

- **NET/ROM read-only "node aware" slice — hear NODES, build a routing table, surface a snapshot (TS↔C# parity)** — the TS parity leg of the C# `Packet.NetRom` library + `NetRomService` node-host wiring ([packet.net#303](https://github.com/M0LTE/packet.net/pull/303), with the LinBPQ-NODES interop proof in [packet.net#305](https://github.com/M0LTE/packet.net/pull/305)). A new `src/netrom/` module makes `@packet-net/ax25` NET/ROM-aware: it parses NODES routing broadcasts heard on the `Ax25Listener` frame-trace tap and maintains a routing table, exposed as an immutable snapshot. It is **strictly read-only** — no TX, no L4 circuits, no NODES origination, no interlinks (those are the real Phase-9 body on the C# side, deliberately out of scope here too). **No SDL, hand-written.** NET/ROM has no SDL figures and no single normative standard — the canonical reference is the original protocol appendix and **BPQ/LinBPQ is the de-facto reference, an interop target, *not* reference truth** — so this is hand-written (it does NOT route through `ax25sdl`), with the same "spec-compliant by default, pragmatism is a named flag" discipline the AX.25 side uses. **Wire codec** (`src/netrom/nodes-broadcast.ts`, `callsign.ts`): `parseNodesBroadcast(info, options?)` decodes a NODES info field — `0xFF` signature + 6-byte sender alias + up-to-11 × 21-byte destination entries (`[7] shifted dest callsign [6] dest alias [7] shifted best-neighbour [1] best quality`), totally and read-only (arbitrary bytes return `null`, never throw); the callsign fields reuse the library's own `readAddress` shifted-callsign codec (one source of truth for the shift/SSID semantics) with a NET/ROM-local tolerance for an all-space ("empty") best-neighbour slot, the alias fields are plain space-padded ASCII. **Divergences are named `NetRomParseOptions` flags** mirroring the C# record — `allowTrailingPartialEntry` (a padded final frame / RF-clipped tail: parse the whole entries, drop the short remainder) and `allowEmptyDestinationList` (a header-only broadcast) — surfaced as the `NETROM_PARSE_STRICT` / `NETROM_PARSE_LENIENT` / `NETROM_PARSE_BPQ` / `NETROM_PARSE_XROUTER` presets (Bpq/Xrouter alias Lenient today — XRouter's "British notion of quality" divergence is a *routing*-table concern, not a wire-parse one; the bytes parse identically). The default ingest path is lenient (read-only promiscuous ingest of third-party broadcasts wants resilience), strict rejects at the byte boundary. **Routing table** (`src/netrom/routing-table.ts`, `quality.ts`): `NetRomRoutingTable` implements the canonical processing heuristics — the originator becomes a directly-heard neighbour at the configured default-port path quality with an assumed direct route (heuristics 3+4); each advertised destination is a route via that neighbour at the multiplicative `combineQuality` `(bq×pq + 128) / 256` decay (heuristic 5); a `trivial-loop guard` zeroes any route whose advertised best-neighbour is our own callsign (heuristic 6, never kept); ≤ 3 routes/destination kept best-first (heuristic 7); a MINQUAL floor + the always-drop-quality-0 rule (heuristic 8); a `maxDestinations` cap (heuristic 9); and an OBSINIT/obsolescence `sweep()` that decrements every route each tick, purges at 0, and drops orphaned neighbours. All floors/caps are canonical-default `NetRomRoutingOptions` knobs (default-port quality 192, MINQUAL 0, OBSINIT 6, 3 routes/dest, 1024 destinations) — never a baked-in BPQ-ism. The C# table gates every mutation/snapshot under a lock (its `FrameTraced` fires on pump threads); the TS table needs none — JavaScript's single-threaded event loop means a synchronously-built snapshot can never observe a torn state — and injects a `now` clock (the TS analogue of `TimeProvider`) for deterministic last-heard stamps + decay. **The read-only tap** (`src/netrom/service.ts`): `NetRomService` is the node-level consumer — `attachPort(portId, myCall, listener)` subscribes the listener's `onFrameTraced` event, which fires for every parsed inbound frame **before** address filtering, so a NODES broadcast (addressed to the literal callsign `NODES`, *not* to us) is heard even though `dispatchInbound` later drops it as not-for-us; the handler gates on RX-direction + UI + PID 0xCF + the `NODES` destination, parses, and ingests — it reads only, sends nothing, posts nothing into any session, and is wrapped in its own try/catch on top of the listener's per-subscriber isolation. `snapshot()` is the public read API (the TS analogue of the C# `INetRomRoutingView` / `Nodes` console command); `sweep()` ages the table (the embedder drives it from a timer — the library stays free of ambient timers); `detachPort` / `dispose` follow the live port set. The whole surface is exported from the package index (`NetRomService`, `NetRomRoutingTable`, `parseNodesBroadcast`, the presets, the model types, `combineQuality`). **Tests (mirror `Packet.NetRom.Tests` + the NET/ROM-aware integration tests):** `tests/netrom/quality.test.ts` (the canonical-formula table + the research-doc worked examples — 200-quality link ≈ 156 at two hops, ≈ 78 at three — + the monotonic-per-hop loop-safety invariant + the valid-byte sweep), `tests/netrom/nodes-broadcast.test.ts` (field decode, the 11-entry cap, the strict-rejects/lenient-accepts pairs for trailing-partial-entry + empty-list, the Bpq/Xrouter presets, the all-space-neighbour tolerance, short-input + 500-case random-garbage totality), `tests/netrom/routing-table.test.ts` (neighbour/direct-route, combined-quality learning, trivial-loop guard, the 3-best cap, in-place re-advertisement, OBSINIT decay/purge/refresh + orphan-neighbour drop, the MINQUAL floor incl. a re-advertisement decaying below it, the destination cap, snapshot ordering), and `tests/netrom/service.test.ts` (the slice end-to-end through a real `Ax25Listener` over `LoopbackTransport`: hears a NODES broadcast on the tap and learns the routes; **observation-only** — no session is built for the broadcaster and the listener transmits nothing; **the read-only guarantee** — a 5-frame NODES storm mid-QSO leaves the live inbound session Connected and still delivering I-frames; disabled-service-hears-nothing; detach-keeps-the-table; a wrong-signature NODES frame is ignored). ## [0.2.0] — 2026-05-16

Adds the `Ax25Listener` class — `@packet-net/ax25` can now act as an inbound-accepting node, the single most valuable parity gap with the C# runtime. Plus two bug-fix carries from the C# runtime that were never quite right in TS either.

### Added

- **`Ax25Listener`** — first-class inbound-acceptance coordinator. Owns one `Ax25Transport`, address-filters inbound frames against `myCall`, dispatches to the per-peer `Ax25ListenerSession` (creating one on first contact — inbound SABM or outbound `listener.connect(remote)`), surfaces `sessionAccepted` + `frameTraced` events, and runs an LRU cache of cached sessions keyed by peer callsign (default cap 64). - **Per-peer session cache** — sessions survive disconnect; the same `Ax25ListenerSession` instance is handed back on reconnect, preserving SRT/T1V smoothing and any out-of-SDL context state (e.g. `sentIFrames` between reconnect attempts).
- **Handler-exception isolation** (#140 carry-over) — every `sessionAccepted` / `frameTraced` subscriber is invoked inside a try/catch; exceptions are routed to a configurable `onHandlerError` sink (default `console.error`) and never escape the inbound pump. A buggy consumer cannot DoS the modem.
- **`Callsign`-aware `connect()`** on the listener — initiate an outbound SABM against a peer using the same per-peer cache; resolves to the session once DL-CONNECT-confirm arrives.
- **SABME factory + classifier branch** — `frame.ts` exports a `sabme(...)` factory and `classify(...)` recognises control byte 0x6F. The full mod-128 sequence-number path remains gated on the `version_2_2` predicate (still effectively false in this runtime); the addition is purely so listener tests can inject SABME and exercise figc4.1's t16 branch.

### Fixed

- **#141 carry-over: via-chain reversal on responses.** `ActionDispatcher`'s outbound frame builders now reverse the inbound trigger's digipeater chain when emitting UA / DM / RR / RNR / REJ / I / UI responses. A peer behind a two-hop digipeater chain (`[GB7CIP, MB7UR]`) sending SABM now gets UA back via the reversed chain (`[MB7UR, GB7CIP]`), per AX.25 v2.2 §C.2 Path Construction. Previously the UA had no via-chain on the wire, so the digi closest to the responder never forwarded it.
- **#143 carry-over: cache-miss DM for non-SABM frames.** When the listener receives a non-SABM frame addressed to us from a peer with no cached session (DISC / RR / RNR / REJ / SREJ / I / FRMR / XID / TEST), it now builds a transient Disconnected session, dispatches the event so the SDL's per-event arm fires (DISC → t13 → DM; everything else → t05 all_other_commands → DM), then drops the transient session without caching it. Previously these frames were silently dropped at the cache-miss filter, leaving the peer's half-open session view hanging.

### Test count

- Unit tests +30 across `Ax25Listener.test.ts` (5), `Ax25ListenerConcurrency.test.ts` (7), `Ax25ListenerMultiPeer.test.ts` (7), `Ax25ListenerRejectAndEdge.test.ts` (10), and `ActionDispatcherViaChain.test.ts` (1) — 1:1 mirror of the 30 C# listener tests.
- Integration tests +3 across `listener-netsim-multi-peer.test.ts` (2) and `listener-linbpq-initiates.test.ts` (1).

### Known limitations

- Per-peer cache is single-threaded (JS is); the C# listener uses a `ConcurrentDictionary` for the same role. The TS surface is event-driven on a single event loop, so the C# concurrency primitives are unnecessary.
- The listener-owned session class is exported as `Ax25ListenerSession` to avoid name-collision with the existing `Ax25Session` from `Ax25Stack`. Functionally similar (both wrap an `SdlSessionDriver`) but the API surfaces differ — `Ax25Session` exposes outbound-flavoured `connect` / `write` / `disconnect`; `Ax25ListenerSession` exposes raw `postEvent` + signal subscription.

## [0.1.0] — 2026-05-16

First public release. Covers AX.25 v2.2 connected-mode happy-path interop with LinBPQ, Xrouter, rax25, and direwolf-style KISS-TCP listeners. Published to npmjs.com from the self-hosted CI runner.

### Added

- **Frame codec** for U-frames (SABM, UA, DISC, DM, UI), S-frames (RR, RNR, REJ), and I-frames. Mod-8 only.
- **`Callsign`** value type — 0-6-char base + 0-15 SSID. `Callsign.parse("M0LTE-7")` round-trips.
- **`Ax25Address`** record + codec — 7-octet on-the-wire callsign with SSID + C/H + E-bit handling.
- **KISS framing** — `KissDecoder` (stateful FEND/FESC/TFEND/TFESC decoder) and `encodeKiss(...)`. Multi-port nibble supported.
- **`Ax25Transport`** — 3-method interface (`start` / `send` / `stop`) — the seam future transports plug into.
- **`WebSerialKissTransport`** — KISS over a `SerialPort` for Chromium browsers.
- **`TcpKissTransport`** — KISS over a TCP socket for Node.js. Reachable via the `@packet-net/ax25/tcp-transport` subpath import so browser bundlers don't pull in `node:net`.
- **`Ax25Stack`** + **`Ax25Session`** — public connected-mode API. `connect({ from, to })` returns a `Promise<Ax25Session>` once SABM/UA completes; the session exposes `onData(cb)`, `onDisconnected(cb)`, `write(bytes)`, and `disconnect()`.
- **Table-driven session machine** — the driver in `src/sdl/session-driver.ts` imports `DataLinkDisconnected`, `DataLinkAwaitingConnection`, `DataLinkAwaitingConnection22`, `DataLinkConnected`, and `DataLinkAwaitingRelease` transitions from the [`ax25sdl`](../../ts-spec/) package, evaluates guards via `src/sdl/guard-evaluator.ts`, executes action chains via `src/sdl/action-dispatcher.ts`, and advances state. Same architecture as the C# runtime in `src/Packet.Ax25/Session/`.
- **T1 retry** on SABM, DISC, and outstanding I-frame, capped at N2 (default 10) via the SDL `RC_eq_N2` guard.
- **First live interop** — integration test against LinBPQ over net-sim (`tests/integration/linbpq-via-netsim.test.ts`) — SABM/UA → I-frame round-trip → DISC/UA, full wire-log evidence captured in [`docs/plan.md`](../../docs/plan.md) amendment log.

### Changed

- Renamed package from `@packet-net/ax25-ts` to `@packet-net/ax25`. Pre-publish rename; no released versions of the `-ts` name exist on npm.

### Known limitations

- **k=1 single-outstanding-I-frame window** — throughput will be SRT-bound on real links.
- **REJ/SREJ recovery loops** — wire frames emit, but the recovery subroutines are no-op stubs (figc4.7 paths not yet walked).
- **Dynamic T1V** — `Select_T1_Value` is a stub; caller-supplied `t1Ms` is honoured statically.
- **No FRMR generation / handling** — inbound FRMR silently dropped.
- **No mod-128 (SABME)** — the SDL `version_2_2` predicate returns false.
- **`via` digipeater paths** — frame factories and codec round-trip them, but `stack.connect({ via: [...] })` throws.
- **No AGW / AXUDP / audio transports** — Web Serial + TCP only.
- **No inbound connection acceptance** — SABM addressed to us with no matching outbound session is silently dropped.
